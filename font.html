Rule — when to track

Letter-spacing is a correction, not a default. Apply it in exactly two cases: tighten the large Playfair display tier (h1–h3, hero) at −0.03em so the big serif doesn't read loose; open up small uppercase labels (eyebrows, kickers, tags) at +0.08em so the caps don't read cramped. Everywhere else — body, leads, buttons, h4–h6 — leave it normal and trust the font's own metric.

Spacing

An 8-step, 8px-based scale, --space-*, in rem (scales with the user's font size). Drives padding, gaps, margins, and section rhythm. Bar = real size.

--space-2xs0.25rem · 4px
--space-xs0.5rem · 8px
--space-sm1rem · 16px
--space-md1.5rem · 24px
--space-lg2.5rem · 40px
--space-xl4rem · 64px
--space-2xl6rem · 96px
--space-3xl8rem · 128px

Rule — snap, don't invent

Reach for the nearest token instead of a fresh pixel value — gaps, margins, padding, section rhythm all pull from this one scale. Two deliberate exceptions are left as-is because snapping them would shift component proportions: component-internal padding that lands on 12px (.btn 0.75rem 1.75rem, .form__input, .review-card__head gap) — 12px sits between --space-xs and --space-sm with no token. If 12 / 32 / 48 start repeating across pages in the Phase-2 rollout, add them then — by data, not upfront.

.btn--icon — icon-only PROPOSED · unifies bespoke .dr-carousel__arrow / .nav__toggle

States — default · hover · focus-visible · disabled forced for reference — hover & focus also work live

Usage

check Do

One .btn--primary per view — the single most important action. Pair it with a .btn--ghost for the alternative.

close Don't

Don't stack two solid buttons side by side, and don't use a button where a .text-link belongs (inline, low-emphasis navigation).

Principles & open items

Voice: editorial, minimal, premium, calm — generous whitespace, restrained colour, one accent. Below: deliberate deviations and the Phase-2 worklist.

Two dark colours coexist. --color-text (#111) drives body text, buttons, and the footer; the nav uses a separate warm #302820 / #322A22 / #B19981 set that isn't tokenised. Consolidate, or formalise as brand-warm tokens.
text-dim fails AA for body text. --color-text-dim (#888) is ~3.3:1 on the light canvas — restrict it to large text and non-essential meta, never body copy.
Elevation isn't tokenised. ~15 one-off box-shadow values exist in styles.css; the --shadow-sm/md/lg scale here is a proposal to replace them.
Card variants aren't unified. .proc-card, .case-card, .review-card, .contact-path each re-implement the card recipe — fold them onto a single .card + modifiers.
--font-icon is Material Symbols Outlined (not "Rounded"); no --fw-bold (700) token — neither family loads 700; no global p/ul/li block-margins — the site spaces with flex gap.
Phase 1 — typography system applied to the homepage + shared components (header, nav, footer, buttons, cards, forms, global sections). Component recipes on this page mirror styles.css; where a class name differs the styles.css name is noted, and anything not yet shipped is tagged PROPOSED. Legacy aliases --font-serif / --font-sans still resolve. index_old.html is excluded.
